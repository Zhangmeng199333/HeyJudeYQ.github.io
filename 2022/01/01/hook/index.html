48、用hook处理组件内定时器

2022-01-01

vue 项目使用 hook 处理组件内定时器的方式。一般是使用生命周期钩子函数beforeDestory来销毁定时器。从Vue源码的lifecycle.js中发现了一种拓展方法，可以提升代码的简洁性，而且还可以减少data里无用变量的定义。

一、使用生命周期函数处理定时器

通常处理组件内定时器的步骤是下面这样的

export default{
    mounted(){
        this.timer = setInterval（() =>{
            // do something
        },1000）
    },
    beforeDestory(){
        clearInterval(this.timer);
    }
}

二、使用 hook 处理定时器

export default{
    mounted(){
        const timer = setInterval(() => {
            // do something
        },1000);
        this.$once('hook:beforeDestory',()=> clearInterval(timer);)
    }
}

三、适用场景:

处理可视化图表的时候，我们需要在数据渲染到页面之前让页面有 loading 动画，mounted 挂载之后停止 loading，beforeUpdate 时开始 loading，updated 之后停止 loading。

使用 hook 处理:

<v-chart>
  @hook:mounted="loading = false" @hook:beforeUpdated="loading = true"
  @hook:updated="loading = false" :data="data"
</v-chart>

四、补充说明：如何实现父组件监听子组件生命周期？

1、方法一：$emit

父组件给子组件传递多个回调函数，依赖子组件在自身的生命周期李通过vm.$emit的方式，调用父组件生命的回调函数。

<!-- 父组件 -->
<template>
    <div>
        <Child @mounted="onMounted"
            @updated="onUpdated"
            @beforeDestory="beforeDestory"
        >
        </Child>
    </div>
</template>

子组件：

mounted(){
    this.$emit('mounted')
},
updated(){
    this.$emit('updated')
},
beforeDestory(){
    this.$emit('beforeDestory')
}

2、方法二：@hook

父组件模板

<template>
    <div>
        <Child @hook:mounted="onMounted"
            @hook:updated="onUpdated"
            @hook:beforeDestory="beforeDestory"
        >
        </Child>
    </div>
</template>

vue源码的lifecycle.js中，Vue的生命周期各个阶段都会去调用一个callHook函数，这个函数支持2个参数：实例vm和生命周期钩子名称,callHook里面执行了vm.$emit(‘hook:’+hook)。

lifecycle.js源码片段（这个文件在Vue源码的src/core/instance/lifecycle.js）

// callHook函数
export function callHook (vm: Component, hook: string) {
  // #7573 disable dep collection when invoking lifecycle hooks
  pushTarget()
  const handlers = vm.$options[hook]
  const info = `${hook} hook`
  if (handlers) {
    for (let i = 0, j = handlers.length; i < j; i++) {
      invokeWithErrorHandling(handlers[i], vm, null, vm, info)
    }
  }
  if (vm._hasHookEvent) {
    vm.$emit('hook:' + hook)
  }
  popTarget()
}

@hook原理：

当我们在子组件上传了对应的@hook:mounted钩子，也就执行了vm.$on(‘hook:mounted’),vue实例在生命周期里本身就会执行vm.$emit(‘hook:mounted’),也触发了我们绑定给子组件的回调函数。
